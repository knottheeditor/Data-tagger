## ğŸš€ 3. How to Process a Batch (Step-by-Step)

### Step 1: Power Up

1. **Turn on your RunPod** (the RTX 4090 instance).
2. **Wait for it to boot** (approx. 30 seconds).

### Step 2: Ingest Content

1. **In the GUI, type the folder you want** into the **TARGETED CLOUD SCAN** box (e.g., `Content-Archive`).
2. **Click INJECT FROM CLOUD**. New files will appear in the list as `pending`.

### Step 3: Verify (The "Look Before You Leap")

1. **Click a file in the list**.
2. **Check the PROJECTED DEST** in the right panel. Does the naming look correct based on your standards?

### Step 4: Simulated Flight (Dry Run)

1. **Check the DRY RUN MODE box**.
2. **Click COMMAND: START BATCH**.
3. **Wait a few minutes** as the worker processes the AI analysis.
4. **Click ğŸ”„ SYNC CLOUD STATS**. The files should now show as `dry_run_complete`.

### Step 5: Live Execution

1. **Once you are happy with the Dry Run results, Uncheck DRY RUN MODE**.
2. **Click COMMAND: START BATCH again**.
3. **The worker will now perform the actual S3 movements**.
